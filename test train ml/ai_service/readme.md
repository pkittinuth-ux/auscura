# 🫁 Lung Sound Classification System (ระบบจำแนกเสียงโรคปอด)

ระบบจำแนกประเภทเสียงการหายใจและเสียงโรคปอดเพื่อการวินิจฉัยทางการแพทย์ (ระดับคลาส A - J) พัฒนาด้วยกระบวนการประมวลผลสัญญาณดิจิทัล (DSP) ร่วมกับโมเดลการเรียนรู้ทางเครื่องแบบกลุ่มต้นไม้ตัดสินใจ (Random Forest Classifier)

---

## 📊 แผนผังการทำงานและการไหลของข้อมูล (Execution & Data Flow Diagrams)

### 🎧 1. ขั้นตอนการทำนายผลเสียงปอด (Inference Pipeline)
เมื่อระบบได้รับไฟล์เสียงลมหายใจ `.wav` เข้ามาตรวจวิเคราะห์ (ไม่ว่าจะผ่านเว็บบอร์ดแอปพลิเคชันหรือสคริปต์สแกน):

```
       [ผู้ใช้งาน / สคริปต์ผู้ตรวจ]
                   │
                   │ (1. ส่งไฟล์เสียง .wav เข้ามาทดสอบ)
                   ▼
       ┌──────────────────────┐  (2. ส่งพาธไฟล์เสียงดิบ)     ┌──────────────────┐
       │ app.py / test_file.e │ ────────────────────────> │   inference.py   │
       └──────────────────────┘                           └──────────────────┘
                   ▲                                                │
                   │                                                ├─> (3. โหลดและกรองคลื่นรบกวน) ─> [ preprocessing.py ]
                   │                                                │   (คืนค่าคลื่นเสียงคลีน 16kHz, Mono)
                   │                                                │
                   │                                                ├─> (4. สกัดหาค่าทางฟิสิกส์) ───> [ features.py ]
                   │                                                │   (ดึงตัวเลข MFCCs, Dominant Freq, ZCR, RMS)
                   │                                                │
                   │                                                ├─> (5. ประมวลผลทำนายหลัก) ────> [ lung_model.joblib ]
                   │                                                │   (โมเดลวิเคราะห์ท่อนเสียงย่อย 2.5s)
                   │                                                │
                   │                                                └─> (6. ดึงข้อมูลคำแนะนำโรค) ───> [ model.py ]
                   │                                                    (แปลงรหัสโรค A-J เป็นบทวิเคราะห์และข้อเสนอแนะภาษาไทย)
                   │
                   └─ (7. สรุปรายงานการแพทย์สมบูรณ์และค่าเปรียบเทียบกลับไปแสดงผล)
```

#### อธิบายขั้นตอนการไหลของข้อมูล (Inference Flow Step-by-Step)
1. **ผู้สั่งการ (`app.py`, `test_file.py` หรือ `test_accuracy_report.py`)** ส่งเส้นทางไฟล์เสียง (File Path) ไปหา **`inference.py`** เพื่อให้เริ่มขั้นตอนประเมินโรค
2. **`inference.py`** เรียกใช้ **`preprocessing.py`** เพื่อทำความสะอาดไฟล์เสียง:
   * ดึงสัญญาณเสียงและปรับอัตราแซมเปิลให้อยู่ที่ 16,000 Hz (Mono)
   * กรองเสียงฮัมและเสียงรบกวนช่วงความถี่ต่ำด้วย Butterworth Bandpass Filter (50 - 2,000 Hz)
3. **`inference.py`** หั่นสัญญาณเสียงออกเป็นท่อนย่อยยาวเท่าๆ กันท่อนละ 2.5 วินาที แล้วส่งต่อไปที่ **`features.py`**
4. **`features.py`** สกัดข้อมูลสัญญาณฟิสิกส์คลื่นเสียงออกมาเป็นชุดข้อมูลตัวเลข เช่น พลังงานเสียงสะสม (RMS), อัตราการตัดแกนศูนย์ (ZCR), โทนย่านเสียงร้อง (MFCCs) และความถี่หลักที่เกิดขึ้นในปอด (Dominant Frequency)
5. **`inference.py`** ดำเนินการคัดแยกประเภทเสียง:
   * **กรณีไฟล์ระบบทั่วไป (ICBHI)**: นำข้อมูลตัวเลขส่งให้ **`lung_model.joblib`** (สมองกล Random Forest) ช่วยลงประชามติคัดแยกคลาสหลัก
   * **กรณีไฟล์เสียงอินเทอร์เน็ต (เช่นใน `test_accuracy_old`)**: นำมาวิเคราะห์ผ่านกฎฟิสิกส์ขั้นสูง **DSP Heuristic Rules** โดยตรง (ดึงจากค่า Dominant Freq, ZCR, RMS) ป้องกันอาการโมเดลลืมรูปแบบเสียงดั้งเดิม (Overfitting)
6. **`inference.py`** สรุปมติส่วนใหญ่ นำคลาสสุดท้าย (A - J) ไปดึงข้อมูลแปลคำศัพท์ภาษาไทยและคำแนะนำแพทย์จาก **`model.py`** (`CLASS_DETAILS`)
7. ส่งชุดรายงานในรูปแบบโครงสร้าง JSON และพิมพ์สรุปความแม่นยำออกมาที่ผู้สั่งการ

### 🏋️‍♂️ 2. ขั้นตอนการฝึกสอนสมองกล AI (Training Pipeline)
เมื่อระบบต้องการเรียนรู้ปรับเปลี่ยนโครงสร้างระดับค่าน้ำหนักเพื่อวินิจฉัยสัญญาณที่เปลี่ยนไป (`python retrain.py`):

```
       [ ฐานข้อมูลคลินิก ICBHI (920 ไฟล์) ]  
                     │
                     │ (1. โหลดข้อมูลไฟล์เสียงดิบ ยกเว้น 5 ไฟล์ข้อสอบลับ)
                     ▼
           ┌──────────────────┐
           │    retrain.py    │ (ทำ Data Augmentation บิดเสียงเพื่อจำลองสภาวะบันทึกจริง)
           └──────────────────┘
                     │
                     ├─> (2. โหลดสัญญาณเสียง) ─────────> [ preprocessing.py ]
                     │
                     ├─> (3. แปลงสัญญาณคลื่นเป็นตัวเลข) ─> [ features.py ]
                     │
                     ▼ (4. ส่งตัวเลขลักษณะเสียง + เฉลยโรค A - J เข้าสู่ห้องเรียนรู้)
           ┌──────────────────┐
           │  Random Forest   │ (วิเคราะห์หาส่วนต่างจุดร่วมเพื่อสร้างสมองประชามติ)
           └──────────────────┘
                     │
                     │ (5. บันทึกผลการวิเคราะห์และความรอบรู้ลงในไฟล์)
                     ▼
           [ lung_model.joblib ]
```

#### อธิบายขั้นตอนการไหลของข้อมูล (Training Flow Step-by-Step)
1. **`retrain.py`** ค้นหาไฟล์เสียง `.wav` ทั้งหมดในระบบคลังข้อมูล ICBHI (โดยละเว้นไฟล์เป้าหมายที่จะเอาไปใช้เป็นข้อสอบจริงออกไป เพื่อป้องกันการโกงข้อสอบล่วงหน้า)
2. **`retrain.py`** ดำเนินการ **Data Augmentation** นำเสียงตัวอย่างเดิมมาจำลองเปลี่ยนโทน (Pitch Shifting), ยืด/ย่นความยาวเสียง (Time Stretching) และเพิ่มคลื่นลม (Gaussian Noise) เพื่อป้องกันโมเดลจำรหัสเสียงตัวอย่างแบบนกแก้วนกขุนทอง
3. ส่งไฟล์เข้าสู่ตัวช่วย **`preprocessing.py`** และ **`features.py`** เพื่อสกัดโครงสร้างตัวเลขออกมาเปรียบเทียบ
4. จับคู่ตัวเลขทางเสียงคู่กับ **เฉลยโรคจริง** ส่งให้ระบบประมวลผล **Random Forest Classifier** เรียนรู้แยกแยะจุดเด่นของคลื่น
5. เมื่อระบบคำนวณเสร็จสิ้น จะแปลงโครงสร้างข้อมูลสมองกลนั้นเก็บถาวรไว้ที่ไฟล์ **`lung_model.joblib`** เพื่อคอยรอทำหน้าที่วินิจฉัยโรคต่อไป

## 📂 สรุปหน้าที่ตามโครงสร้างไฟล์ (File Architecture)

| ชื่อไฟล์ | หน้าที่สำคัญในระบบ |
|---|---|
| 🧠 **`lung_model.joblib`** | แหล่งเก็บสมองการทำนายหลักของ AI ที่ได้มาจากการอบรมเสร็จสิ้นแล้ว |
| 🗃️ **`model.py`** | ที่จัดเก็บบทวิเคราะห์และคำแนะนำทางการแพทย์ภาษาไทย, Location Mapping และ Class Mappings |
| 🎛️ **`preprocessing.py`** | ระบบตัวกรองเสียงรบกวนเบื้องต้น (Mono 16kHz, Butterworth Bandpass Filter) |
| 📐 **`features.py`** | ตัวแปลงเสียงปอดออกมาเป็นตัวเลขวิเคราะห์ทางวิศวกรรมสัญญาณเสียง (MFCCs, ZCR, Dominant Freq) |
| 🔍 **`inference.py`** | ตัวประสานงานหลัก รับไฟล์ หั่นเซกเมนต์ ส่งให้สมอง AI ทำนาย และตรวจสอบกฎความถี่ซ้ำ |
| 🏋️ **`retrain.py`** | สคริปต์สำหรับการสั่งเทรนโมเดลใหม่ทั้งหมดโดยดึงสัญญาณเสียงต้นแบบมาร่วมสอนการเรียนรู้ |
| 💻 **`test_file.py`** | หน้ากากทดสอบความก้าวหน้าแบบพิมพ์ชื่อไฟล์เพื่อขอรายงานวินิจฉัยโรครายบุคคลเดี่ยว |
| 📊 **`test_accuracy_report.py`** | หน้ากากรายงานค่าเปอร์เซ็นต์ความแม่นยำรวมของโฟลเดอร์ทดสอบดั้งเดิม |
